# Orchestration

Use `skill-smokeplanner` before accepting or publishing a reusable skill.

1. Point the planner at a `SKILL.md` file.
2. Review missing-section findings and risky command warnings.
3. Run only the suggested local commands that are appropriate for the repo.
4. Attach the generated Markdown or JSON to the release-candidate PR.
5. Do not treat the plan as approval for external side effects.

The CLI reads local files only and performs no command execution.
